# #3.1 Definitely Typed

[_GitHub_](https://github.com/leesiyun/react-masterclass/commit/81ea579f7519c40c15f5b8b477dba5ef06b88d0d)

이번 강의에서는 create-react-app에 TypeScript 설치를 하겠습니다.

**2가지 방법**

하나는 지금까지 한 걸 다 지우고 처음부터 다시 시작하는 법입니다.

```bash
npx create-react-app my-app --template typescript
```

create-react-app을 이용해서 새로운 react app를 만들 때 TypeSript가 함께 구성되어있습니다.

전부 지우고 이 명령어를 입력해주면 됩니다.

그리고 styled-component를 재설치한 뒤, 이전 강의에서 했던 theme 설정도 해주면 도됩다

만약 지금까지 했던 걸 지우기 싫다면, 나중에 배우게 될 모든 typescript를 전부 설치해주어야 합니다.

```bash
npm install --save typescript @types/node @types/react @types/react-dom @types/jest
```

터미널의 위 코드를 입력하고 파일의 확장자(extension)를 바꿔줍니다.

TypeScript에서는 .js를 쓰지 않고 .ts를 사용합니다.

그리고 TypeScript와 React에서는 .tsx를 사용합니다.

index.js의 (확장자) 이름을 .tsx로 변경해주고

그리고 App.js도 마찬가지로 App.tsx로 바꿔줍니다.

이제 서버를 재시작합니다.

clear 입력후 npm start

create-react-app은 너무 똑똑하기 때문에 프로젝트 안의 TypeScript를 감지합니다.

이제 에러가 확인되는데

이 에러는 TypeScript가 styled-component가 어떤 것인지 모른다는 것입니다.

TypeScript는 모든 걸 알아야 합니다.

현재 TypeScript는 props가 뭔지 몰라서 에러를 발생시키고 있습니다.

TypeScript에게 도움을 받는건 유용합니다.

에러들이 TypeScript가 일하고 있다는 것입니다.

이제 무엇을 해야할까요?

어떤 라이브러리나 패키지는 TypeScript로 만들어진게 아닌 JavaScript 기반으로 만들어져있습니다.

그리고 이것이 JavaScript를 기반으로 만들어졌다면 TypeScript는 이게 뭔지 모르기 때문에 에러가 발생합니다.

TypeScript는 코드가 실행되기 전에 모든 걸 알 필요가 있습니다.

현재 TypeScript는 코드가 실행되지 전에 에러들을 알려주고 있습니다.

TypeScript로 만들어지지 않은 라이브러리를 import 할 경우에는

“npm install @types/styled-components.” 명령어를 실행해 줍니다.

이 명령어를 실행시켜주고 그 다음 에러들이 해결됐는지 확인합시다.

설치 후에 보면, TypeScript의 에러가 해결되었습니다.

이제 TypeScript는 styped-component가 무엇인지 압니다.

그렇게 때문에 prop들이 무엇인지도 압니다.

lightTheme은 선언되었지만 사용되지는 않았습니다.

이건 에러는 아니고 그냥 좋은 힌트 입니다.

우린 방금 @types/styped-components를 설치했습니다.

@types란 아주 큰 Github repository인데 말하자면 모든 유명한 npm 라이브러리를 가지고 있는 저장소입니다.

그리고 여기서 라이브러리나 패키지의 type definition을 알려줍니다.

우리는 TypeScript가 전부 다 알아야 한다는 건 알고 있습니다.

그런데 가끔 패키지가 TypeScript가 아닌 JavaScript일 경우에는 TypeScript에게 모든 정보를 주지는 않습니다.

그 라이브러리나 패키지를 사용한다면, TypeScript에게 모든 정보를 알려줘야합니다. 물론 시간이 오래 걸리기는 합니다.

그래서 사람들이 모여서 type definition을 만들었습니다.

Type definition이란, styled-component의 소스코드를 보고 TypeScript에게 해 줄 설명을 만들어 내는 것입니다.

npm 에서 @types/styped-components을 살펴보면 TypeScript에서 뭐가 뭔지 알려주는 설명들을 볼 수 있습니다.

그리고 이 @types name space는 많은 사람들이 이용하는 Definitely Typed Repository에서 왔습니다.

어떤 라이브러리나 패키지를 사용하고 싶은데 DefinitelyTyped에 없다면

그리고 그걸 TypeScript에서 사용하고 싶다면 여기다 코드를 push 해주면 됩니다.

복습해봅시다.

styled-component를 위해 type definition을 설치했습니다.

가장 많이 사용할 유명한 것들은 @types/styled-components에서 찾을 수 있습니다

가끔은 사람들이 만들어 놓지 않았다면 여기서 찾을 수 없을 수도 있습니다.

그럼 그 때는 여러분이 만들 좋은 기회라고 생각하면 됩니다.

아니면 이 명령어가 작동하지 않을 때 쓸 수 있는 다른 방법을 다음에 알려드리겠습니다.

하지만 여전히 TypeScript에게 뭐가 뭔지 설명은 해야할 것입니다.

아주 멋진 Definitely Typed 커뮤니티를 이용해서

TypeScript에게 이 package가 뭔지 알려준 걸로 해결할 수 있었습니다.

세팅은 이게 전부 입니다.

아마 다음강의에서도 조금더 세팅을 하겠지만

일단은 Definitely Typed라는 멋진 게 있다는걸 알아둡시다

여기서 TypeScript로 작업할 때 필요한 대부분의 type defintion들을 얻을 수 있다는 것도
