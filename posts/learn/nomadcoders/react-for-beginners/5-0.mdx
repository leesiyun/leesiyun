# 📘
# #5.0 Introduction

<div style="color:gray; font-style:italic;">2022/02/17 목요일</div>

<br />

이번 섹션에서는 Create React App과 친해지는 시간을 가져봅시다.

create-react-app은 리액트 어플리케이션을 만드는 최고의 방식입니다.

이전 강의에 만든 리액트 어플리케이션은 스크립트를 import함으로써 만들어졌습니다.

기본적으로 모든걸 스스로 해야했죠.

그러나 create-react-app을 사용한다면, ReactJS 어플리케이션을 만드는 것이 훨씬 쉬워집니다.

이유는, create-react-app은 엄청나게 많은 스크립트들과 많은 사전 설정들을 대신 준비해주기 때문입니다.

create-react-app는 개발 서버에 접근해주거나, 자동 새로고침 기능,

즉각적으로 어플리케이션 안에 CSS를 포함시켜주는 등 여러 기능들이 있습니다.

<br />

또한 create-react-app은 publish하는 명령어를 갖고 있습니다. 웹사이트를 publish할 준비가 되었다면

<div style="color:gray; font-size:12px;">Terminal</div>

```
npx create-react-app my-app
```

명령어를 이용해 어플리케이션을 실행시켜봅시다

<br />

이 명령어를 실행하기 위해서 미리해야할 것은 NodeJS를 설치해놓는 것입니다.

nodejs.org페이지로 이동해 OS버전에서 맞는 NodeJS를 다운받으십시오.

설치한 후, console 창에서 `node -v` 를 입력해 커맨드가 있는지 확인해봅시다.

정상적으로 버전이 확인된다면 제대로 다운로드 된 것입니다.

`node -v`를 확인했다면 이제 `npx install`을 확인해 봅시다.

이 npx 커맨드도 문제 없이 작동된다면 준비가 끝났습니다.

<br />

이제 Documents 폴더 또는 원하는 폴더에 새로운 프로젝트를 만들어 봅시다.

이전에 만들었던 폴더와 같은 이름의 프로젝트를 만들것이기 때문에 이전에 만들었던 react-for-beginner 폴더는 삭제해두었습니다.

<br />

`npx create-react-app` 뒤에 프로젝트(어플리케이션) 이름을 넣고 실행시켜보십시오.

load가 되고 이것저것 install 된 후, 프로젝트 이름의 폴더가 만들어졌습니다.

이제 VSCode에서 만들어진 폴더를 열어 package.json을 확인합시다.

NodeJS에 대한 지식이 있다면 package.json 안에서 스크립트를 찾아 낼 수 있다는 걸 알고 있을 겁니다.

<br />

package.json안에서 실행시킬수 있는 script들이 들어있는 걸 확인할 수 있습니다.

이것들은 create-react-app에 의해 만들어진 것입니다.

<br />

여기서 우리는 `start` 스크립트를 실행해보겠습니다.

터미널에서 `npm run start` 또는 `npm start`를 입력합니다.

그러면 자동적으로 브라우저가 열리며 development server(개발용) 서버가 만들어집니다.

<br />

이것으로 우리는 아주 멋진 페이지를 얻었습니다.

지금 보이는 페이지가 create-react-app을 사용해서 어플리케이션을 만들었을때의 초기버전입니다.

이제 목록에서 src(폴더명)이 확인 가능해졌습니다.

src 폴더는 모든 파일을 넣을 폴더입니다. 모든게 다 src 폴더 안에 있어야 합니다.

src 폴더 안에는 많은 것이 들어있지만 여기서 가장 중요한건 `index.js`입니다.

`index.js`를 살펴봅시다. ReactDOM, document.getElementById와 같은 아주 익숙한 것들을 볼 수 있을겁니다.

<br />

create-react-app은 어플리케이션이 index.html안에 넣어주도록 설정되있고

이미 `index.html`은 만들어져 있습니다.

<br />

자 이제 `index.html`을 살펴봅시다. `root`가 보일겁니다. 이전 강의에서 다뤘던 바로 그것입니다.

이제는 손수 만들 필요가 없이 이미 설정되어 있는걸 확인할 수 있습니다.

src안에 있는 코드를 가져다가 자동으로 html 페이지의 어딘가로 넣어줍니다.

예를들어 inspect 탭의 코드를 살펴봅시다.

우리가 봤던 것들에 더해서 `static/js~~~` 들이 추가되어있는걸 확인할 수 있습니다.

이것은 실제로 `index.html` 내에는 존재하지 않습니다. 단지 우리의 편의를 위해 넣어진것들입니다.

<br />

 create-react-app을 이용하면 얻을 수 있는 장점 한 가지는 Auto-Reload(자동 재실행)입니다.

예를들어 `app.js`에 있는 `Edit src/App.js~~~` 코드를 “`Hello`”로 바꾸고 저장을 하면 

새로고침하는 순간 “Hello”로 바뀌는걸 알 수 있습니다. 이것은 매우 유용한 기능입니다. 

<br />

사실 우리가 모르는 것들은 제쳐두고 대부분은 거기서 거기입니다.

ReactDOM을 import하고 render, 컴포넌트 하나 getElementById안에 렌더링해주고 있고

App의 안쪽도 살펴 보면 단지 함수 컴포넌트일 뿐입니다. 거기에 약간의 jsx가 들어 있습니다.

그 외 나머지는 우리가 지워야 할 몇몇 기본 코드들만 있을 뿐입니다.

<br />

처음부터 새롭게 시작하기 위해 설정된 CSS나 기타 등등 모두 지워줍시다

그리고 지금은 감당할 수 없는 테스팅도 지워줍니다.

<div style="color:gray; font-size:12px;">React</div>

```jsx
import React from "react";
import ReactDOM from "react-dom";
import APP from "./APP";

ReactDOM.render(
	<React.StrticMode>
		<APP />
	</React.StrictMode>
	document.getElementByID("root")
);
```

<div style="color:gray; font-size:11px;">index.js</div> 

`index.js`에서 위의 코드만 남기고 다 삭제했습니다.

우리가 원하는건 세 개의 import과 ReactDOM.render 입니다.

<br />

이전 강의에 어려운 방법으로 만들어 봤기 때문에 우리는 이것이 무엇인지 알 수 있습니다.

<br />

<div style="color:gray; font-size:12px;">React</div>

```jsx
function App() {
	return (
		<div>
			<h1>Welome back!</h1>
		</div>
	);
}
```

<div style="color:gray; font-size:11px;">App.js</div> 

그리고 `App.js`에서도 CSS, logo 등 필요없는 건 지워줍니다.

그리고 `h1`이 들어있는 `div`를 반환하도록 추가해줍시다.

<br />

이건 단지 컴포넌트이지만 NodeJS로 작업하고 있기 때문에 파일을 각각 분리시키는 것이 가능하고 좀 더 조직적으로 구성 할 수 있게 되었습니다.

한 파일당 한 컴포넌트를 가지고 있고, `index.js`에 그 컴포넌트를 import 시켜주었습니다.

즉, 차후 버튼과 같은 다른 컴포넌트도 한개의 파일로 이루어질거라는 걸 의미합니다.

<br />

 create-react-app에는 꽤 많은 기본파일들이 딸려 있는데 `App.test.js`, `App.css`, `index,css`, `logo.svg`, `reportWebVitals.js`, `setupTests.js` 를 모두 지워줍시다.
 
<br />

이제 깔끔한 create-react-app의 설치판을 얻었습니다.

리액트 어플리케이션을 랜더링해줄 뿐인 `index.js`와  Welcom back!을 랜더링해줄 뿐인 `App.js` 만 남겨졌습니다.

<br />

우리가 해야하는 일은 어플리케이션을 만들고 `npm start`를 실행하는 일뿐입니다.

<br />

이번 섹션에서는 npm을 실행하고 어플리케이션을 깔끔하게 정리했습니다.
