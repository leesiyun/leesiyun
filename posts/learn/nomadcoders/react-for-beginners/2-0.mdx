이제부터 React JS를 기초부터 배워보겠습니다.

React JS가 여러분을 위해 해결해주는 문제를 이해하는 것부터 시작해봅시다.

기술은 도구를 고치기 위해 만들어집니다.

그래서 기술이 고치려고 하는 문제를 이해하면 그 기술을 더 잘 이해할 수 있게 됩니다.

React JS의 포인트를 이해하지 못하고 이를 배우려고 한다면 혼란스러울 수가 있습니다.

실제로 많은 사람들에게 벌어지는 일입니다.

일단 React JS에 달려들어서 학습을 시작합니다 왜 React JS가 만들어졌는지 이해하지 못한 채로 말입니다.

만약 여러분이 뭔가 배우는데, 이것이 어떤 문제를 해결하는지 이해하지 못한다면 해결하려는 문제를 직접 겪어보지 못한다면 그냥 암기를 할 수 밖에 없을 겁니다.

그러고는 ‘왜 이렇게 하는거야? 왜 이렇게 보이는거지? 왜이렇게 복잡해?’’라는 궁금증이 생깁니다.

해결하고자 하는 문제를 제대로 이해하지 못한다면 그 해결책이 이해가 잘 가지 않을 수 밖에 없습니다.

그래서 그 해답을 이해하지 못한 채로 끝나거나 그냥 암기하고 받아 적고 끝납니다.

외우고 외우고 또 외우고

그래서 저는 그렇게 하는 대신에 다른 방법으로 진행해보겠습니다.

바닐라 JS코드와 React JS 코드를 비교해 보는 방식입니다.

지금부터 여러분들께 React JS를 팔아보려고 합니다.

React JS가 여러분을 얼마나 생산적으로 바꿔주는지 보여드리겠습니다.

React JS는 UI를 interactive하게 만들어줍니다.

아주 간단하게 여러분들의 웹사이트에서 interactivity(상호 작용)을 만들어줍니다.

React JS는 말 그대로 이걸 위해 태어났습니다.

React JS 팀은 여러분이 무엇이 필요한지 알고 있고 interactive한 웹사이트를 만들기 위해 무엇이 요구되는지 알고 있습니다.

예를 들어 다음 섹션에서는 화면에 버튼 하나와 텍스트 하나만 있는 아주 간단한 바닐라 JS 앱을 만들어 보겠습니다.

이 어플리케이션에서 버튼을 클릭할 때마다 화면의 텍스트를 업데이트 할 수 있도록 만들어봅시다

클릭을 세는 어플리케이션입니다.

조금 자세히 이야기하자면 클릭할 때마다 화면의 텍스트를 업데이트해서 한번 두번 또는 몇번이든 클릭되었다고 나오는 어플리케이션입니다.

이제 이 어플리케이션이 작동하려면 무엇이 필요한지 생각해봅시다.

무엇보다도 버튼이 필요할 것입니다. HTML에 버튼을 만들고 만든 버튼을 HTML에서 찾기 위해 id나 class를 주고 그리고 javascript에서 그 버튼을 찾아야 합니다.

그래서 document.getElementById나 querySelectore등을 사용할 겁니다.

그 다음으로 button.addEventListener를 입력하고 function을 실행해야 할 겁니다.

이런 많은 단계를 거쳐야 합니다.

<br />

React JS 팀은 이 점을 인지하고 있습니다.

버튼을 생성하는데, 클릭을 감지하는데, 무엇이 필요한지 알고 있고 그래서 그들이 이 작업들에 대한 지름길을 만들었다면?

그것이 바로 React JS입니다. React JS는 아주 좋은 아이디어들을 모아둔 것인데 거의 모든 것이 interactive 개발자들이 필요한 것입니다.

그것을 전부 하나로 모아놨습니다.

여러분이 따라야 할 규칙들이 좀 있지만, 다음 강의에서 React JS 없이 쓴 바닐라 JS코드와 ReactJS로 작성한 코드를 한번 비교해볼건데, 코드를 비교해보는 순간 React JS가 왜 태어났는지, 이것이 얼마나 편리한지 깨닫게 될 것입니다.
