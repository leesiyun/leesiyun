이번 강의에서는 React JS를 배우는 것이 왜 좋은 생각인지 말해보려고 합니다.

그리고 여기에 여러분의 시간과 노력을 투자하는 것이 왜 안전한지도 알아봅시다.

어떤 것을 배운다는 것은, 그것에 베팅해본다라는 의미라고 생각합니다.

개발자의 인생에서 가장 중요한 자원은 바로 시간입니다.

무언가를 배우는데 2달이 걸렸다고 합시다. 그런데 1년이 지나니까 사라졌습니다.

그건 배우기에 좋은 도구가 아니었던 것입니다. 2달을 그냥 낭비해버린것입니다.

만약 무언가를 배우는데 2달이 걸렸는데 그것을 5년 7년 10년 쓸 수 있다고 생각해 보십시오.

그것이 바로 좋은 선택이고, 훌륭한 시간 투자인것입니다.

어떤 베팅은 다른 것보다 더 위험하기도 합니다.

2015년 초에 React JS를 처음 봤을 때에는 조금 위험했습니다.

원칙에 따르면 이건 위험한 선택이었던 것입니다.

출시된지 얼마 안됐고, 아무도 미래를 보장할 수 없었습니다.

페이스북이 React JS를 계속 유지할지 버릴지 아무도 몰랐습니다.

구글이 어떤 제품을 없앨지 계속 유지할지 모르는 것처럼 말입니다.

그러나 그 당시에는 매우 위험했지만 지금은 아주 안전하다고 볼 수 있습니다.

지금은 React JS는 아주 좋은 선택입니다.

<br />

자 이제 왜 그런지 설명하겠습니다.

첫 번째. 신기술을 배울 때 중요한 것은 누가 이 기술을 사용하는지, 그들의 규모가 얼마나 큰지 꼭 살펴봐야합니다.

그리고 이 기술이 그들에게 얼마나 중요한지도 봐야합니다.

<br />

예를들어 상위 1만 개의 웹 사이트 중에서 44.76%는 React JS를 사용한다고 합니다.

전세계 상위 1만개의 제일 큰 웹사이트들 중에서도 엄청 많은 숫자인 것입니다.

<br />

이건 ‘똑똑한 사람들이 소속된 큰 기업들이 사용해서 좋은 것’ 같은 단순한 이유로 좋은 소식이라고 말하려는건 아닙니다.

요점은 React JS를 보면 에어비앤비, 인스타그램, 페이스북, 넷플릭스와 같은 큰 회사들이 사용하고 있습니다.

그럼 스스로에게 물어봅시다. ‘좋아, 그럼 React JS가 그들에게 얼마나 중요하지?’

넷플릭스에게 React JS는 얼마나 중요할까?

React JS가 넷플릭스 사이트를 돌리고 있는데 넷플릭스 웹사이트는 넷플릭스에게 얼마나 중요할까?

정말 중요하다고 말할 수 있을 것입니다.

<br />

페이스북 웹사이트는 페이스북에게 얼마나 중요할까요?

에어비앤비도 마찬가지 입니다.

<br />

그런데 이 회사들 모두가 React JS로 작업한다는 것입니다.

이런 거대한 회사에서 이 분야의 연구를 자세히 하는 사람들(프론트엔드를 크게 바꾸고 싶지 않아하는 사람들)은 이미 큰 규모의 회사에 있고 회사의 매출도 크기 때문에 사이트를 자주 수정하거나, 오늘은 React JS를 해보고 내일은 Angular를 써보고 그럴 시간이 없습니다. 안정적이고, 작동하는 것을 선택해야합니다.

만약 이런 사람들이 사용하는 library나 framework를 여러분이 배우려고 한다면? 그건 아주 좋은 선택이라고 말할 수 있습니다.

<br />

두 번째. React JS가 어디서부터 왔는가입니다. React JS는 페이스북이 만들었고 페이스북은 React JS로 웹사이트 전부를 다시 만들었습니다.

그리고 오늘날도 페이스북은 React JS를 지원하고 있습니다.

React JS로 작업하고, 이를 개선하기 위해 자금을 들여 개발자를 고용해서 React JS를 더 좋은 방향으로 개선하기 위해 투자하고 있습니다.

그리고 우리는 그 코드를 무료로 사용할 수 있습니다.

페이스북이라는 거대기업이 React JS에 투자하고 있는 것입니다.

React JS에 투자도 많이 해왔고, 그들의 웹사이트도 React JS로 다시 만들었는데 이들이 React JS를 금방 폐기할 수는 없을 것입니다.

이것이 두 번째 포인트였습니다.

<br />

세 번째. React JS는 정말 큰 커뮤니티를 가지고 있습니다.

아마 여러분들이 찾아볼 수 있는 커뮤니티 중에서도 가장 큰 편에 속할 것 입니다.

왜냐하면 React JS는 Javascript와 매우 가깝기 때문입니다.

이에 대해서는 나중에 React JS가 왜 어썸한지 설명할 때 다뤄보겠습니다.

React JS는 Javascript와 아주 가깝기 때문에, 여러분이 React JS로 작업할 때 대부분의 작업은 그냥 일반 Javascript에 대한 작업일 겁니다.

그래서 React JS는 Javascript의 커뮤니티를 거의 다 빌려온 셈인데, 이 커뮤니티가 워낙 거대합니다.

요점은 React JS 자체는 작고, 대부분의 경우 React JS는 그냥 Javascript이기 때문에, 그리고 React JS를 배우고 작업하는 것은 여러분을 더 나은 Javascript 개발자가 되도록 하기 때문에 그래서 커뮤니티가 아주 거대합니다.

책이나 코스의 양, 그리고 library들이나 가이드, package, 가르쳐줄 사람, 직업, 채용 등 엄청 많고 거대합니다.

이건 아주 중요한 일입니다. 기술을 배울 때 그 기술에 대한 생태계가 존재하는 것이 좋습니다.

예를 들어, React JS를 기반으로 제작된 framework들도 있는데

여러분의 인생을 편하게 해줄 framework를 만드는 사람들이 Reack JS에 기반해서 framework를 만들고 있다는 것입니다.

<br />

이건 React JS를 기반으로 다른 것을 제작하는 생태계가 존재한다는 것입니다.

아직 React Native는 이야기도 안했습니다.

페이스북 팀은 React Native라는 도구도 가지고 있는데 이건 iOS나 안드로이드 어플을 React JS 코드로 만들 수 있게 해줍니다.

이것이 바로 React JS를 한 번 배우면 어디든 쓸 수 있다고 말하는 이유 입니다.

React JS는 확장하면서 한계를 밀어 붙이고 있습니다. 또한 React JS로 VR 경험을 제작하려는 미친 사람들도 있습니다.

<br />

지금까지 살펴봤듯이 정말 많은 것들이 React JS로 진행되고 있습니다.

생태계가 폭팔적으로 늘어나고 있고, 페이스북이 지원하고 있습니다.

그리고 많은 거대한 웹사이트 들이 React JS 뒤에 있습니다.

그래서 오늘날 React JS를 배운다는 것은 정말 좋은 선택입니다.
